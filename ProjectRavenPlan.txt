Project Raven - Revised Phased Plan (v2)
Core principle: Local-first. The free tier is a complete, standalone desktop app. Backend is layered on top for the paid tier.

Phase A: Local-First Foundation
Goal: Electron app shell with persistent local storage, plan mode switch, and two-window architecture

A1 -  Project scaffold: Electron + React + TypeScript + Tailwind (already done from Phase 1), add electron-store for local persistence
A2 - Plan mode system: Global config mode: 'free' | 'pro' stored locally. All conditional UI/logic branches from this single flag. Default: free.
A3 - Local API key management: Onboarding flow asks for Deepgram + Anthropic keys. Keys encrypted and stored via electron-store. Validated on entry (ping each API). Persisted across restarts. Never asked again unless user edits in settings.
A4 - Two-window manager: Main process manages Dashboard (normal window, light theme) + Overlay (floating, frameless, stealth-capable). IPC bridge between them.
A5 - Dashboard shell: Header with "My Raven" + Mode dropdown + Detectable/Undetectable toggle + "Start Raven" button. Empty session list placeholder. Light theme.
A6 - Overlay shell: Pill toolbar (logo, hide, mic/stop) + expandable response panel + input bar with quick action chips. Dark/translucent theme. Stealth mode (setContentProtection).

Checkpoint: App launches with two windows. User enters API keys once during onboarding, never again. Keys persist across restarts. Both windows render with proper themes.

Phase B: Core Engine (Audio -> Transcript -> AI)
Goal: The actual product functionality, all running locally using user's API keys

B1 - Audio capture: Microphone capture via Web Audio API (16kHz mono). Device selection. Visual recording indicator.
B2 - Deepgram transcription: WebSocket streaming to Deepgram using user's stored key. Interim + final results. Speaker diarization.
B3 - Claude AI integration: Send transcript + active mode prompt to Claude using user's stored key. Streaming response display.
B4 - Overlay wiring: Record button starts audio -> streams to Deepgram -> transcript shows in overlay. Quick action chips (Assist, "What should I say?", Follow-up, Recap) each send different prompt templates.
B5 - Stealth mode: macOS setContentProtection(true), Windows SetWindowDisplayAffinity. Toggle in dashboard header + overlay pill. Persisted in settings.
B6 - Global hotkeys: Cmd/Ctrl+Enter (AI suggestion), Cmd/Ctrl+Shift+H (toggle overlay), Cmd/Ctrl+Shift+R (toggle recording).

Checkpoint: Full working product. User can record a meeting, see live transcription, get AI suggestions, all while invisible to screen sharing. This is the MVP.

Phase C: Sessions
Goal: Session history with local persistence

C1 - Session model (local): Sessions stored in electron-store or local SQLite (id, title, transcript, aiResponses, duration, mode, startedAt, endedAt).
C2 - Session lifecycle: Start -> recording + transcription -> Stop -> auto-save locally.
C3 - Auto-naming: After session ends, send transcript to Claude -> generate title -> update session.
C4 - Session history (dashboard): List grouped by date. Each row: title + duration + timestamp + three-dot menu (Copy, Regenerate title, Delete).
C5 - Session detail view: Click session -> full transcript + AI responses. Back/forward navigation.
C6 - Session export: Export as Markdown or plain text.

Checkpoint: Sessions persist locally. Dashboard shows history grouped by date. User can review, rename, delete, export sessions.

Phase D: Modes System
Goal: Custom AI prompt profiles

D1 - Modes model (local): Modes stored locally (id, name, prompt, isActive, files, notesTemplate).
D2 - Manage Modes modal: Left sidebar (mode list + New Mode + Raven Templates) + right pane (name, prompt textarea, file upload, notes template).
D3 - Mode CRUD: Create, rename (click-to-edit), set active (blue checkmark), delete.
D4 - Built-in templates: Sales, Recruiting, Team Meet, Looking for Work, Lecture - each with pre-built prompt + notes sections.
D5 - Notes template builder: Add sections, edit, remove, reorder with drag handles.
D6 - Mode ↔ Session integration: Active mode's prompt sent as system prompt to Claude. Mode dropdown in dashboard header.
D7 - File upload as context: Attach files to a mode -> stored locally -> sent as context to Claude.

Checkpoint: Modes fully functional. User creates custom AI profiles, switches between them, sessions use active mode's prompt.

Phase E: Settings
Goal: Full Cluely-style multi-tab settings

E1 - Settings window/modal: Two-column layout. Sidebar: General, Keybinds, Profile, Language, API Keys*, About. Bottom: Quit Raven. (API Keys tab visible only in free mode)
E2 - General tab: Detectable banner + toggle, open on login toggle, theme (Light/Dark/System), version + check for updates, mic source selection + test, Advanced accordion.
E3 - Keybinds tab: All shortcuts in sections (General, Window, Scroll) with keycap badges. Click to remap.
E4 - Profile tab: Display name, avatar (stored locally in free mode). Password/security section hidden in free mode.
E5 - Language tab: Transcription language dropdown, output language dropdown.
E6 - API Keys tab (free mode only): Deepgram key input + validate, Anthropic key input + validate, usage/status indicators, "Keys are stored locally and encrypted" reassurance text.
E7 - Toast notifications: Slide-in from top-right, loading -> success/error, auto-dismiss.

Checkpoint: Settings complete. Free users manage API keys here. All preferences persist.

Phase F: Overlay Polish
Goal: Match Cluely's interaction quality

F1 - Collapsible states: Collapsed (pill + input) vs expanded (pill + response + input). Smooth animation.
F2 - Response formatting: "Assist" badge, "Say this next:" quoted text, markdown rendering.
F3 - Screenshot feature: Capture screen region, attach to AI request.
F4 - Keyboard-driven movement: ⌘+arrows to reposition overlay, ⌘+Shift+arrows to scroll response.
F5 - Clear conversation: ⌘+R to reset context.
F6 - System tray: Tray icon with context menu (Show/Hide, Start/Stop, Settings, Quit).

Checkpoint: Overlay feels premium and matches Cluely's UX.

Phase G: Backend + Paid Tier
Goal: Backend API for paid users, auth, cloud features

G1 - Backend scaffold: Node.js + Express/Fastify + PostgreSQL + Prisma. Health check endpoint.
G2 - Auth system: Sign up, login, JWT + refresh tokens, bcrypt. Google OAuth.
G3 - Plan management: Free vs Pro flag on user model. Stripe integration for billing.
G4 - Proxy API routes: Backend proxies Deepgram + Claude calls using our API keys for Pro users. Free users never hit our backend for AI calls.
G5 - Cloud sync: Sessions, modes, settings sync for Pro users. Conflict resolution (last-write-wins or merge).
G6 - Auth in Electron: Login/signup screens in dashboard (Pro mode). Store JWT locally. Auto-refresh.
G7 - Billing tab (Pro mode only): Replaces API Keys tab. Shows plan, pricing, upgrade card with gradient.
G8 - Calendar integration (Pro): Google Calendar OAuth, sync meetings, show in dashboard.
G9 - Team features (Pro): Shared modes, team workspace, invite members.

Checkpoint: Paid tier fully functional. Free tier completely unaffected - no backend dependency.

Phase H: Distribution
Goal: Ship it

H1 - App icons (all sizes, Raven branding)
H2 - electron-builder config: DMG (Mac), NSIS (Windows)
H3 - Auto-updater: electron-updater + GitHub Releases
H4 - Code signing: Mac notarization, Windows signing
H5 - Backend deployment (for Pro tier): Railway/Fly.io/AWS
H6 - Landing page + docs


Phase I: FE Polish (Handoff to FE Engineer)
Goal: Production-grade visual quality

Animations, transitions, micro-interactions
Loading skeletons
Edge cases, responsive adjustments
Final visual review


Total: 9 phases, ~55 tasks
Key difference from v1: Phases A–F build a fully standalone, no-backend product. Phase G layers on the backend and paid tier. Free users never need a server, never need an account, never see billing UI.
