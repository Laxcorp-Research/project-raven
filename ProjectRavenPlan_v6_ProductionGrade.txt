5. OS-NATIVE AEC: Use each platform's battle-tested voice processing (Apple/Microsoft).

PHASE C: Sessions (Persistent History) ✅ COMPLETE
================================================================================
Goal: Session history with robust local persistence

C1 - Session Data Model ✅
  - Local SQLite database via better-sqlite3 (synchronous, reliable)
  - Schema: sessions (id, title, transcript_json, ai_responses_json, mode_id, 
    duration_seconds, started_at, ended_at, created_at)
  - Indexes on started_at and created_at for fast sorting
  - Migration system for schema updates
  - Located: src/main/services/database.ts

C2 - Session Lifecycle ✅
  - Start: Create session record with started_at
  - During: Append transcript entries and AI responses in memory
  - Stop: Write final data, calculate duration, set ended_at
  - Auto-save: Periodic flush to disk during long sessions (every 60s)
  - Crash recovery: Recover in-progress session on restart
  - Located: src/main/services/sessionManager.ts

C3 - Auto-Naming ✅
  - After session ends, send transcript summary to Claude
  - Generate concise title (3-7 words, max 50 chars)
  - Strict prompt with XML tags to avoid conversational responses
  - Fallback: timestamp-based name if Claude fails ("Session at 3:45 PM, Feb 6")
  - Regenerate title available via three-dot menu

C4 - Dashboard Session List ✅
  - Sessions grouped by date (Today, Yesterday, This Week, Earlier)
  - Each row: title, duration badge, timestamp, entry count
  - Click to expand: full transcript + AI responses inline
  - Three-dot menu for actions
  - Located: src/renderer/src/components/dashboard/SessionList.tsx

C5 - Session Detail View ✅
  - Inline expansion (click row to expand/collapse)
  - Full transcript with You/Them formatting (cyan/gray bubbles)
  - AI responses shown in purple section
  - Copy transcript to clipboard
  - Export as Markdown file (includes metadata, transcript, AI responses)
  - Delete session (with confirmation)

C6 - Session Playback ⏳ DEFERRED (Future Enhancement)

LESSONS LEARNED (Updated)
================================================================================

1. OS-native voice processing (Apple/Microsoft) is superior to manual AEC
   - Battle-tested, hardware-optimized
   - Handles timing automatically

2. better-sqlite3 requires externalization in vite config
   - Native modules conflict with ESM bundling
   - Solution: rollupOptions.external: ['better-sqlite3']

3. Claude title generation needs strict prompting
   - Without constraints, responds conversationally
   - Use XML tags + examples + explicit "output ONLY" instructions

4. Session lifecycle needs careful timing
   - Start session BEFORE audio capture begins
   - End session AFTER final transcript entries saved
   - Auto-save protects against crashes

5. Inline expansion works better than separate detail view
   - Users can quickly scan multiple sessions
   - Context preserved (no navigation away)
